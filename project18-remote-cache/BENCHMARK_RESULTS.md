# 缓存性能测试结果

## 测试环境
- 项目：4 个 C++ 库 + 1 个可执行文件
- 系统：Linux
- Bazel 版本：7.x

## 实际测试数据

### 测试 1: 无缓存（基准）
```
Elapsed time: 2.464s
Real time: 3.888s
```

### 测试 2: 磁盘缓存（写入）
```
Elapsed time: 2.535s
Real time: 4.143s
```
写入缓存略慢，因为需要额外的 I/O 操作。

### 测试 3: 磁盘缓存（读取）
```
Elapsed time: 0.381s
Real time: 1.562s
```

## 性能对比

| 测试场景 | 构建时间 | 加速比 |
|---------|---------|--------|
| 无缓存 | 3.89s | 1.0x (基准) |
| 磁盘缓存(写) | 4.14s | 0.94x |
| 磁盘缓存(读) | 1.56s | **2.49x** |

## 结论

- **磁盘缓存读取提供 2.5x 加速**
- 在大型项目中，加速比会更明显（可达 10x-100x）
- 缓存对于 CI/CD 和团队协作至关重要

## 为什么小项目加速不明显？

这个示例项目很小，编译时间本身就短。在实际大型项目中：
- 编译时间可能是几分钟到几小时
- 缓存命中率更高
- 加速效果更显著

## 运行测试

```bash
./benchmark.sh
```
